## 一、完整功能模块

### 1. 程序/文件夹一键封禁

用户选择：

```
D:\Apps\SomeSoftware\
```

工具自动扫描里面的：

```
.exe
.dll 调用的子程序
updater.exe
service.exe
helper.exe
crashreporter.exe
```

然后生成规则：

```
禁止该目录下所有 exe 出站联网
禁止该目录下所有 exe 入站联网
```

这是基础版核心。

------

### 2. 进程树追踪

很多软件不是主程序联网，而是拉起子进程联网。

例如：

```
MainApp.exe
 ├─ Helper.exe
 ├─ Update.exe
 ├─ ServiceHost.exe
 └─ CrashReporter.exe
```

工具需要实时监听进程创建事件：

```
父进程在封禁名单中
↓
子进程自动加入封禁名单
```

这样可以解决“我明明禁了主程序，还是能联网”的问题。

------

### 3. Windows 防火墙规则管理

第一阶段可以直接调用系统防火墙，不必上来就写驱动。

用 PowerShell / netsh / Windows Defender Firewall API 创建规则：

```
New-NetFirewallRule `
  -DisplayName "AppNetGuard_Block_xxx" `
  -Direction Outbound `
  -Program "D:\Apps\xxx.exe" `
  -Action Block
```

每个 exe 建立：

```
出站阻止规则
入站阻止规则
```

并且给规则统一命名，方便删除和恢复。

------

### 4. 服务识别与管控

很多软件会安装 Windows 服务。

需要扫描：

```
services.msc
```

重点识别：

```
服务路径在目标文件夹内
服务名称包含目标软件名
服务启动的 exe 被封禁
```

功能：

```
查看相关服务
禁用服务联网
可选：停止服务
可选：禁用服务自启动
```

注意：不要乱禁系统服务，例如：

```
svchost.exe
lsass.exe
wininit.exe
services.exe
explorer.exe
System
```

这些应加入系统白名单。

------

### 5. 计划任务识别

很多软件通过计划任务偷偷启动更新器。

扫描：

```
taskschd.msc
```

识别任务动作里包含：

```
目标文件夹路径
目标软件名
updater
update
helper
launcher
```

功能：

```
列出相关计划任务
可选禁用
可选只阻止任务启动的 exe 联网
```

------

### 6. 实时联网监控

显示当前有哪些进程正在联网：

```
进程名
PID
程序路径
远程 IP
远程端口
协议 TCP/UDP
连接状态
是否已被拦截
```

可以用：

```
GetExtendedTcpTable
GetExtendedUdpTable
ETW
Windows Filtering Platform
```

第一版建议先用 `GetExtendedTcpTable`，简单稳定。

------

### 7. 规则组/配置档案

用户可以创建配置：

```
Adobe 全套封禁
游戏启动器封禁
某个离线软件封禁
开发工具仅允许局域网
```

每个配置包含：

```
{
  "name": "Example App Block",
  "paths": ["D:\\Apps\\Example"],
  "programs": ["example.exe", "helper.exe"],
  "services": [],
  "scheduledTasks": [],
  "mode": "block_all"
}
```

------

### 8. 联网模式

建议做 4 种模式：

```
允许全部
禁止全部
仅允许局域网
仅允许指定域名/IP
```

其中最实用的是：

```
禁止全部
仅允许局域网
```

比如某些软件只需要访问：

```
127.0.0.1
192.168.x.x
10.x.x.x
172.16.x.x
```

就可以允许局域网，禁止公网。

## 二、技术路线建议

### 第一阶段：MVP 可用版

目标：先做出能用的版本。

技术栈：

```
C# + WPF / WinUI 3
.NET 8
PowerShell 调用防火墙
SQLite 保存规则
管理员权限运行
```

功能：

```
选择 exe / 文件夹
扫描所有 exe
一键创建防火墙阻止规则
一键恢复
显示当前规则
```

这是最容易落地的版本。

------

### 第二阶段：增强版

加入：

```
进程树监听
服务扫描
计划任务扫描
联网连接查看
规则组管理
日志记录
```

进程监听可以用：

```
WMI Win32_ProcessStartTrace
```

或更高级的：

```
ETW
```

------

### 第三阶段：专业版

加入：

```
Windows Filtering Platform
实时拦截
按进程路径动态拦截
域名/IP 规则
局域网白名单
托盘常驻
通知提醒
```

这一阶段才考虑 WFP。

WFP 更强，但复杂度明显高很多。前期不建议直接上。

------

## 三、核心架构

```
AppNetGuard
├─ UI 层
│  ├─ 首页
│  ├─ 程序扫描
│  ├─ 规则组
│  ├─ 实时连接
│  └─ 日志
│
├─ Core 核心层
│  ├─ AppScanner
│  ├─ FirewallManager
│  ├─ ProcessWatcher
│  ├─ ServiceScanner
│  ├─ TaskSchedulerScanner
│  └─ NetworkMonitor
│
├─ Data 数据层
│  ├─ SQLite
│  ├─ RuleProfile
│  └─ RuleHistory
│
└─ Agent 后台层
   ├─ 托盘常驻
   ├─ 自动补规则
   └─ 开机自启
```

## 四、最推荐的开发顺序

```
1. 做 UI：选择文件夹 / exe
2. 扫描目标目录下所有 exe
3. 调用防火墙创建阻止规则
4. 保存规则到 SQLite
5. 支持一键解除封禁
6. 加入服务扫描
7. 加入计划任务扫描
8. 加入实时联网监控
9. 加入进程树自动拦截
10. 最后再考虑 WFP
```

## 五、最小可行功能清单

第一版只要做到这些，就已经有价值：

```
选择文件夹
扫描所有 exe
批量禁止联网
批量恢复联网
显示已封禁程序
导出/导入规则

```

然后第二版再做：

```
检测相关服务
检测计划任务

子进程自动封禁
```

## 六、关键注意点

不要盲目禁这些系统进程：

```
System
svchost.exe
services.exe
lsass.exe
wininit.exe
csrss.exe
explorer.exe
RuntimeBroker.exe
SearchHost.exe
```

否则可能导致：

```
系统网络异常
应用商店异常
登录异常
更新异常
局域网异常
```

所以你的工具必须有：

```
系统白名单
规则回滚
操作前备份
一键恢复所有规则
```
